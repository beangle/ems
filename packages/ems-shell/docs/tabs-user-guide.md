# EMS 工作台标签页 — 使用说明

本文面向 **EMS 门户最终用户**，介绍内容区上方 **多标签工作台** 的用法：如何打开、切换、刷新、固定与关闭标签，以及标签栏右侧工具按钮的含义。

> 若你负责开发或维护 ems-shell，行为约定与实现细节见 [workspace-tabs.md](./workspace-tabs.md)（设计说明）。

---

## 1. 什么是工作台标签页？

在 EMS 门户中，从 **左侧菜单**（或顶部应用分组）点击功能入口后，页面不会整页跳转，而是在 **工作台** 中打开一个 **标签页**：

- **标签条**：显示当前已打开的功能名称，位于内容区顶部；
- **内容区**：显示该功能对应的页面（iframe 嵌入页、微前端子应用或门户首页）；
- **工具区**：标签条最右侧，提供「标签操作」下拉与「全屏」按钮。

你可以同时打开多个标签，在几个功能之间快速切换，类似浏览器标签，但仍在同一门户窗口内。

---

## 2. 打开标签

### 从菜单打开

1. 在左侧菜单中找到功能项并 **单击**；
2. 若该功能尚未打开，工作台会 **新建一个标签** 并加载对应页面；
3. 若 **同一入口已经打开过**，不会重复新建，而是 **直接切换到已有标签**（去重）。

### 首页标签

若门户配置了欢迎页，多标签模式下会自动出现 **「首页」** 标签（带 🏠 图标）：

- 始终排在 **最左侧**；
- **不可关闭**，相当于固定标签；
- 用于快速回到门户首页。

### 在新浏览器标签中打开

若菜单项本身配置为「新窗口打开」（`target="_blank"`），则走浏览器新标签，**不经过**工作台。

---

## 3. 切换标签

| 操作 | 说明 |
|------|------|
| **单击标签标题** | 切换到该标签，显示对应内容 |
| **← / → 方向键** | 在标签条上移动焦点（需先用 Tab 键聚焦到某个标签） |
| **Home / End** | 跳到第一个 / 最后一个标签（仅移动焦点） |
| **Enter 或空格** | 激活当前获得焦点的标签（与单击切换效果相同） |
| **浏览器后退 / 前进** | 若之前通过切换标签产生了浏览历史，可用浏览器后退/前进回到曾激活的标签 |

切换后，当前标签会自动 **滚入可视区域**（标签很多、需要横向滚动时尤其有用）。

---

## 4. 刷新标签

有时页面数据过期、提交后需要重载，或子页面状态异常，可以 **刷新当前标签**：

| 方式 | 操作 |
|------|------|
| **双击标签标题** | 在标题文字区域双击（不要点在右侧 × 或 📌 上） |
| **右键菜单** | 在标签上 **右键** → 选择 **「刷新」** |
| **工具栏菜单** | 点击标签条右侧 **▼** → 选择 **「刷新」**（针对 **当前激活** 的标签） |

### 刷新会做什么？

刷新会按 **打开该标签时的入口地址** 重新加载内容，相当于「从菜单重新进入一次」。

需要注意：

- 刷新 **不会** 改变标签上显示的名称；
- 若你在标签内的页面里又点链接跳到了子页面，刷新会回到 **最初从菜单打开的那一页**，而不是停留在子页面；
- 这与浏览器里「刷新 iframe 内当前页」不同，是 EMS 工作台的有意设计，以保证标签名称、菜单高亮和恢复逻辑一致。

---

## 5. 关闭标签

| 方式 | 操作 |
|------|------|
| **关闭按钮 ×** | 点击标签右侧的 **×** |
| **右键菜单** | 右键标签 → **「关闭」** |
| **关闭其他** | 右键标签 → **「关闭其他」**（保留当前标签及所有 **已固定** 的标签） |

**不可关闭** 的情况：

- **首页** 标签；
- 你已 **固定** 的标签（见下一节）。

关闭最后一个普通标签后，工作台可能仍保留首页等固定标签。

---

## 6. 固定标签

常用功能可以 **固定**，避免误关，并使其在标签条上 **靠前显示**（紧跟首页之后）。

| 操作 | 说明 |
|------|------|
| **固定** | 右键标签 → **「固定标签」**；或点击未固定标签右侧的 × 区域（部分场景下 × 会先变为固定） |
| **取消固定** | 右键 → **「取消固定」**；或点击标签右侧的 **📌** 图标 |
| **固定后的 ×** | 固定后，右侧按钮变为 **图钉 📌**，点击可取消固定；右键菜单中 **「关闭」** 对固定标签不可用 |

固定标签在 **「关闭其他」** 时会被保留。

---

## 7. 右键菜单与工具栏菜单

在 **标签标题区域** 上 **右键**，或在标签条右侧点击 **▼**，可打开操作菜单（两者选项相同；▼ 菜单作用于 **当前激活** 的标签）：

| 菜单项 | 说明 |
|--------|------|
| **刷新** | 按入口地址重新加载（见第 4 节） |
| **在新标签打开** | 在 **浏览器新标签页** 中打开该标签记录的入口 URL |
| **固定标签 / 取消固定** | 切换固定状态（首页无此项） |
| **关闭** | 关闭该标签（固定标签、首页无此项） |
| **关闭其他** | 关闭除当前标签与所有固定标签外的其他标签 |

**说明：**

- 右键菜单 **只在标签条上** 出现；在下方内容区右键不会弹出此菜单；
- 按 **Esc** 或点击页面其他区域可关闭右键菜单。

---

## 8. 全屏显示

点击标签条右侧的 **⛶（展开）** 按钮，可将 **整个工作台**（标签条 + 内容区）全屏显示，便于专注操作。

- 全屏后图标变为 **压缩**，再次点击退出全屏；
- 也可使用浏览器自带的退出全屏（如 **Esc**）。

---

## 9. 标签很多时怎么办？

### 横向滚动

标签超出屏幕宽度时，标签条支持 **左右滑动 / 滚动** 查看全部标签。切换到某个标签时，它会自动滚到可见位置。

### 数量上限

系统对同时打开的标签数有上限（默认 **30** 个，由管理员配置）。达到上限后再打开新菜单项时，会自动关闭 **最右侧、未固定** 的一个标签以腾出位置。

### 建议使用方式

工作台适合 **少量标签并行** 使用，而不是像浏览器一样堆积大量标签。建议：

- 及时 **关闭** 暂时不用的标签；
- 对常用入口使用 **固定**；
- 需要时从 **侧栏菜单** 或 **菜单搜索** 重新进入，而不是长期保留几十个标签。

目前 **不提供**「全部标签列表」「左右翻页按钮」等额外导航控件；请依赖滚动、键盘切换和关闭/固定来管理标签。

---

## 10. 刷新浏览器后标签还在吗？

**在同一次浏览器会话内**（未关闭浏览器标签页 / 窗口），工作台会把已打开的标签保存到 **会话存储**（sessionStorage）。因此：

- **刷新门户页面（F5）** 后，通常会 **恢复** 之前打开的标签及最后激活的那一个；
- **关闭浏览器标签页或窗口** 后，会话结束，下次进入门户会 **重新从菜单开始**（除非 URL 中带有特定菜单 hash，由门户启动逻辑决定）。

退出登录时，门户会清理导航相关状态。

---

## 11. 标签名称与地址栏

- 标签上显示的名称，来自 **菜单项标题**（打开时确定）， **不会** 随标签内页面继续跳转而自动改名；
- 浏览器 **地址栏** 会随 **你切换的标签** 更新，但 **不会** 跟随 iframe 内部的每一次链接跳转；
- 因此，标签名表示的是「从哪个菜单入口进来的」，而不是「当前子页面标题」。

了解这一点，有助于正确使用 **刷新** 和 **关闭其他** 等功能。

---

## 12. 单标签模式（若管理员启用）

部分部署可能关闭多标签（`multiTab=false`）：

- 标签条 **隐藏**，视觉上像单页应用；
- 每次从菜单打开新功能会 **替换** 当前内容，而不是并排多个标签；
- 本文所述的右键菜单、固定、关闭其他等 **多标签能力不可用**。

默认情况下 EMS 门户 **启用多标签**，即本文描述的行为。

## 13. 常见问题

**Q：单击标签没反应，要等一下才切换？**  
A：单击有约 250ms 延迟，用于与 **双击刷新** 区分。若需快速切换，可连点或使用键盘 Enter。

**Q：双击标签没刷新？**  
A：请双击 **标题文字区域**，不要点在 × 或 📌 按钮上。

**Q：刷新后为什么不是我在子页面看到的内容？**  
A：刷新加载的是 **菜单入口 URL**，见第 4 节。

**Q：同一个菜单点了两次，为什么只有一个标签？**  
A：同一入口 **自动去重**，第二次会激活已有标签。

**Q：右键内容区为什么没有「刷新/关闭」？**  
A：这些操作仅针对 **标签条上的标签项**，请在标签标题上右键。
